## Overview

This project is a research-grade prototype of an **AI-powered self-healing network**. It combines:

- **Random virtual network topology** — new graph generated every run using Prufer sequence algorithm
- **Live packet capture** — Scapy sniffs real traffic on the network interface
- **Hybrid 3-layer IDS** — Signature engine + Random Forest + IsolationForest
- **Confidence-based automated response** — Block, Reroute, or Log
- **Self-healing routing** — Dijkstra on a weighted NetworkX graph
- **RL-inspired edge weights** — edges penalised on attack, rewarded on safe traffic
- **Adaptive learning** — model retrains automatically on confirmed attack samples
- **Live web dashboard** — real-time topology, animated packets, attack alerts

---

## Architecture

```
Live Packet Capture (Scapy on eth0 / Mininet interface)
        ↓
Feature Extraction (13 flow-level statistics per 2s window)
        ↓
Hybrid Detection System
   ├── Signature Engine   (instant pattern matching)
   ├── ML Classifier      (Random Forest — 150 trees, 13 features)
   └── Anomaly Detector   (IsolationForest — zero-day detection)
        ↓
Confidence-Based Decision Engine
   ├── ≥ 80%  →  BLOCK   (IP banned + iptables + Dijkstra reroute)
   ├── 50–79% →  REROUTE (switch penalised + path recomputed)
   └── < 50%  →  LOG     (recorded, traffic allowed)
        ↓
RL Weight Update (penalise/reward edges)
        ↓
Adaptive Learning (retrain every 50 confirmed attacks)
        ↓
Live Dashboard  →  http://<WSL2-IP>:8050
```

## How Detection Works

Three methods run in priority order per 2-second flow window:

**1. Signature Engine** (fastest)
```
syn_flood:   syn_count > 15  AND  packet_rate > 10
port_scan:   unique_dst_ports > 8  AND  packet_rate > 8
brute_force: unique_dst_ports == 1  AND  packet_count > 25  AND  avg_pkt_size < 200
icmp_flood:  proto_tcp_ratio < 0.1  AND  packet_rate > 10
```

**2. Random Forest ML Classifier**
- 150 decision trees, trained on 12,000 synthetic flow samples
- 4 classes: Normal / DoS-DDoS / Port Scan / Brute Force
- Returns label + confidence percentage

**3. IsolationForest Anomaly Detector**
- Trained only on normal traffic
- Catches zero-day attacks not seen in training data

**Decision table:**

| Confidence | Action |
|---|---|
| ≥ 80% | BLOCK — IP banned + iptables + reroute |
| 50–79% | REROUTE — switch penalised + path recomputed |
| < 50% | LOG — recorded, traffic allowed |
| ≥ 85% (non-signature) | LEARN — pattern saved to `learned_signatures.json` |

## How Routing Works

- Network modelled as a weighted NetworkX graph
- Random topology generated every run via **Prufer sequence** algorithm
- **Dijkstra** finds minimum-weight path from h1 to h2
- On attack: attacker's switch edges penalised (`weight += 10.0`)
- Dijkstra reruns → picks new cheapest path avoiding penalised switches
- On safe traffic: edges slowly rewarded (`weight -= 0.5`, min 1.0)
- At least 2 node-disjoint paths guaranteed at startup

## Configuration

All settings in `config.py`:

```python
FLOW_WINDOW        = 2.0   # seconds per flow analysis window
CONFIDENCE_BLOCK   = 80    # >= this % → BLOCK
CONFIDENCE_REROUTE = 50    # >= this % → REROUTE
CONFIDENCE_LEARN   = 85    # >= this % → learn new signature
EDGE_PENALTY       = 10.0  # weight added to attacker's switch edges
EDGE_REWARD        = 0.5   # weight removed from safe nodes per cycle
NUM_SWITCHES_MIN   = 4     # minimum switches in random topology
NUM_SWITCHES_MAX   = 8     # maximum switches in random topology
NUM_ATTACKERS_MIN  = 2     # minimum attacker hosts
NUM_ATTACKERS_MAX  = 4     # maximum attacker hosts
```
